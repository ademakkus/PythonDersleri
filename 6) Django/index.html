1. Heroku Login adresinden, üyelik başlatıyoruz.
  2. Heroku Start adresinden python dilini seçerek devam ediyoruz.
  3. Sonraki adımlara geçerek, işlemim sistemine göre, setup dosyasını indiriyoruz
  4. kurulum bittikten sonra, command Promp ekranı açarak, heroku login komutunu çalıştırıyoruz.
  5. heroku create "uygulama isminiz"
  6. pip install --user pipenv komutu cmd üzerinde çalıştırarak, makinemize virtualenv yüklüyoruz. Eğer, daha pip kurmadıysanız,
    1. Pip indir adresinden kodları kopyalayıp, bir get-pip.py adında bir dosyaya kaydediniz. Dosya masa üsütnde yer alsın.

      if(python dosayı açmayı bilmiyosanız == True): masa üstünde bir txt dosyası açıp, kopyaladığınız kodları içerisine yapıştırım file(dosya) menüsünden save as (farklı kaydet) seçeneğini seçerek, adını get-pip.py olarak değiştiriniz. Not : uzantısının .py olduğundan emin olunuz. else: devam edin :)

  7. Django Kurulumu
    1. Django Download adresine giderek, ilgili kod satırını kopyalıyoruz. pip install Django
    2. cd Desktop masaüstüne geliyoruz.
    3. mkdir DjangoProject DjangoProject adında bir klasör oluşturuyoruz.
    4. cd DjangoProject komutu ile o klasör içerisine girip,
    5. django-admin startproject "heroku içerisinde oluşturduğunuz uyulamanın(app) adını veriniz."
    6. cd projeadı komutunu çalıştırınız ve projenin dizini içerisine giriniz.
    7. python manage.py runserver komutu uygulamanızı ayağa kaldırır ve Siteyi Aç aderesinden sitenizi görüntüleyebilirsiniz.
  8. İlk App uygulaması
    1. App Oluşturma adresinden yönergeyi takip ederek ilk ekranınız yazınız.
    2. Not : cmd ekranında iken, dir anahtar kelimesini ekranda çalıştırırsanız o anda bulunduğu alan içerisindeki dosyaları listeler. python manage.py komutunu kullanırken mutlaka, manage.py dosyasının bağlı bulunduğu dizin içerisinde olmanız gerekmektedir. python: can't open file 'manage.py': [Errno 2] No such file or directory hatasını alıyorsanız cmd ekranında hangi dizinde olduğunuzu kontrol ediniz :)
    3. python manage.py startapp categories komutu ile, categories adında bir app oluşturduk.
    4. categories app içerisinde urls.py adında bir dosya ekleyiniz ve içeriğini düzenleyiniz.
    5. categories ve home app'lerini oluşturduktan sonra, default url home olarak belirledik ve sonrasında herolu'ya publish edildi
  9. Heroku publish
    1. Heroku Setup ilk olarak bu adreste yer alan yönergeleri takip ediyoruz.
    2. cd Desktop cd(current directory) komutu ile masaüstüne geliyoruz. cd ProjeKlasoru cd "Masa üstünde oluşturduğunuz klasöre geliyoruz" virtualenv app yeni bir enviroment oluşturuyoruz,
      Not : Eğer, virtual env oluşturmada hata alıyorsanız.
      pip uninstall virtualenv pip install virtualenv virtualenv app
    3. Virtual enviromet'a geçiş yapmak için yeni bir cmd (command promp - terminal(mac için)),
      Windows için app\Scripts\activate
      Mac için source app\activate
    4. Procfile adında uzantısız bir dosya oluşturup içerisine web: gunicorn myproject.wsgi ekledik. myproject kısmına projenizin adını yazınız
    5. enviroment aktif iken, pip install gunicorn paketini indiriyoruz ve paketi pip freeze ile inip inmediğini kontrol ediniz.
    6. Be sure to add gunicorn to your requirements.txt file as well.
      eklediğiniz paketlerin requirements.txt de içerisinde yer aldığından emin olunuz dosyanınz yok ise, proje içerisinde olduğundan emin olun (manage.py) ile aynı dizinde olduğunuzdan, cmd ekranında enviroment aktif olmalıdır.
      pip freeze > requirements.txt derseniz, dosya otomatik olarak oluşacaktır. hata alırsanız elle ekleyiniz.
    7. pip install django-heroku paketini ekliyoruz.
      Be sure to add django-heroku to your requirements.txt file as well. txt dosyasına eklemeyi unutmayın.
    8. import django_heroku satırını ana proje içerisinde yer alan settins.py dosyası içerisinde en üst satıra ekleyiniz.
    9. # Activate Django-Heroku. django_heroku.settings(locals()) setting.py içerisinde en alt satıra ekliyoruz.
    10. yeni bir cmd ekranı proje içerisine geliniz ve (manage.py)) git init
    11. git add *
    12. git commit -m "First initialize"
    13. heroku git:remote -a muratvuranok
    14. git push heroku master
